# SEO Audit Workflow

Use for public landing pages, marketing sites, blogs, docs, product pages, and pre-launch checks.

## Checks

- Unique title and description per public route.
- Open Graph and Twitter metadata.
- Canonical URL where relevant.
- Robots and sitemap setup when app has public indexed pages.
- Semantic heading order.
- Descriptive links and image alt text.
- Structured data when it directly fits the page type.
- Core Web Vitals risks: blocking assets, oversized images, unnecessary client JS.
- Public content is crawlable without auth.

## Output

Group findings as:

- `Block launch`
- `Fix before public launch`
- `Improve later`

Do not invent schema markup if the page type does not justify it.
