# Debug Workflow

Use for bugs, failing tests, runtime errors, broken flows, or regressions.

## Rules

- Investigate before fixing.
- Reproduce the failure when possible.
- Trace data/control flow to the source.
- Change the smallest responsible surface.
- Add or update a regression test when feasible.
- Run verification before declaring fixed.

If three fixes fail, stop and summarize evidence, hypotheses, and blockers.
