# Self-Optimization Preferences
tags: self-learning optimization ruvector sona

## Adopt From RuVector

- Record trajectories for important agent runs.
- Promote successful repeated patterns.
- Use quality scores and user feedback to influence future routing.
- Consolidate noisy short-term lessons into durable patterns.
- Keep the system inspectable and markdown-first.

## Avoid For MVP

- Heavy Rust/GNN integration.
- Opaque vector DB as source of truth.
- Automatic model fine-tuning without explicit evidence gates.
- Cloud shared brain dependencies for private project memory.

## Default Gates

- Never learn secrets, PII, tokens, or customer data.
- Failed patterns can be stored only as avoid-list lessons.
- Promote only patterns with good quality or repeated success.
